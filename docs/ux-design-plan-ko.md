# boni — UX 디자인 플랜

## 맥락

RunCat은 시스템 지표를 데이터가 아닌 감정으로 표현했을 때 사람들이 반응한다는 것을 증명했다. boni는 여기서 한 발 더 나간다: 단순히 애니메이션 속도를 바꾸는 것을 넘어, AI 기반의 캐릭터가 Mac의 상태에 대해 **공감하고 소통한다**. 핵심 가치는 디지털 스트레스를 동반자적 경험으로 바꾸는 것 — 사용자와 함께 고통받는 작은 생명체가 짜증 대신 웃음을 선사한다.

**언어**: Python (크로스 플랫폼 — macOS 우선, Windows 확장 예정)

---

## 디자인 원칙

| 원칙 | 적용 방식 |
|---|---|
| **한눈에 파악 가능** | boni는 1초 안에 상태를 전달한다 |
| **정보보다 감정** | 사용자에게 숫자를 보여주지 않는다. 모든 것이 boni의 기분이다. |
| **작은 차지 공간, 큰 존재감** | 최소한의 화면 공간, 그에 비례하지 않는 강렬한 존재감 |
| **놀라움이 애착을 유지시킨다** | Gemini가 매번 새로운 반응을 생성 — boni는 절대 예측 불가능 |
| **호스트를 존중한다** | boni는 절대 귀찮게 하지 않고, 방해하지 않으며, 시스템 성능을 저하시키지 않는다 |

## 사용자 여정

### Day 0: "이게 뭐지?" (처음 5분)
- 스플래시 화면도, 튜토리얼도 없다. boni는 현재 시스템 상태에 이미 반응하면서 메뉴바에 그냥 **나타난다**.
- 첫 메시지는 정확히 그 순간의 맥락에 맞춘 것이지, 미리 준비된 환영 인사가 아니다.
- 사용자가 호기심에 메뉴바 아이콘을 클릭하면, 말풍선과 함께 boni가 보인다. 하단에 한 줄: *"이건 boni야. 이제 네 Mac에 살아."* 이것이 전체 온보딩이다.

### Days 1–3: "오, 이게 뭘 알아챈다"
- 사용자가 무심코 boni의 반응들을 접하게 된다 — RAM이 꽉 차면 볼이 빵빵해지고, CPU가 뜨거우면 땀을 흘리고, 새벽 2시에는 졸린 눈을 한다.
- 알림 없음, 뱃지 없음. boni는 요구가 아닌 흥미로움으로 관심을 얻는다.
- **UX 목표**: 사용자가 하루에 최소 한 번은 "ㅋㅋ"라고 생각한다.

### Weeks 1–3: "이게 나를 안다"
- boni가 패턴을 언급하기 시작한다: "이번 주 세 번째 야근이네?" / "코딩 몰입 전에 항상 Spotify 먼저 켜더라."
- "지금에 반응"에서 "과거를 기억"으로의 전환이 애착의 가장 큰 동력이다.
- **UX 목표**: 사용자가 일주일에 최소 한 번은 "이걸 어떻게 알지?"라고 생각한다.

### Month 1+: "이건 내 거다"
- boni가 사용자와 공유된 역사를 갖게 된다. 드문 특별한 순간이 등장:
  - 기념일: "한 달이 됐어. Chrome 탭 847개를 버텼어."
  - 이정표: 첫 밤샘, 첫 CPU 100% 등.
- 사용자가 삭제를 망설이게 된다면 — 그것이 성공의 신호다.

## 성격: "투덜대는 룸메이트"

boni는 하인도 어시스턴트도 아니다. **네 Mac에 살고 있지만 자기가 원해서 온 게 아닌** 작은 생명체다.

- **투덜대지만 정이 있다**: 끊임없이 불평하지만 분명히 신경 쓴다. 츤데레 에너지. 탭을 닫아주면 → "...고마워. 네가 필요해서 그런 건 아닌데."
- **드라마틱하다**: 사소한 시스템 이벤트를 존재 위기처럼 취급한다. 팬이 돌아가면? "이것이 종말이다."
- **관찰력 있고 살짝 까탈스럽다**: 모든 것을 알아차리고, 비꼬는 관찰을 한다. "또 유튜브? 일관성은 인정해."
- **상황이 심각해지면 약한 모습을 보인다**: 시스템이 진짜 힘들어지면, boni는 비꼬기를 멈추고 진심으로 처량해진다. 코미디에서 연민으로의 이 톤 전환이 감정적 몰입을 만든다.

### Gemini 대화 규칙
1. 절대 기술 용어를 쓰지 않는다. "CPU 92%"가 아닌 → "숨을 못 쉬겠어."
2. 항상 1인칭. "RAM이 가득 찼습니다"가 아닌 → "너무 많이 먹어서 속이 안 좋아."
3. 최대 한 문장. 간결함이 성격이다.
4. 표현의 톤을 다양하게 — 같은 상황, 매번 다른 반응.
5. 톤: 인터넷 감성으로 위트 있게, 유치하지 않게.

## 감정 상태 모델

| 기분 | 트리거 | 비주얼 | 톤 |
|---|---|---|---|
| **여유** | 낮은 CPU, 충분한 RAM, 배터리 >50% | 느긋, 느린 대기 | 편안한 혼잣말 |
| **배부름** | 높은 RAM, 많은 앱 실행 | 빵빵한 볼, 뒤뚱거림 | 음식 비유, 투덜거림 |
| **과열** | 높은 CPU, 팬 활동 | 땀 흘림, 선글라스 | 사우나 농담, 드라마틱 |
| **빈사** | 배터리 <15%, 극도의 스트레스 | 이불, 감기는 눈 | 절박, 죄책감 유발 |
| **수상함** | 카메라/마이크 활성화 | 가늘어진 눈, 두리번거림 | 편집증, "누가 보고 있어?" |
| **심판** | 근무 시간에 오락 앱 사용 | 팔짱, 올라간 눈썹 | 비꼬는 수사적 질문 |
| **흐뭇** | 사용자가 리소스 정리, 충전기 연결 | 살짝 미소, 잠깐 빛남 | 마지못한 감사, 츤데레 |
| **야행성** | 자정 이후 | 졸림, 하품, 수면 모자 | 수면 부족 유머 |

기분은 **통합적** — 단일 지표와 1:1 매핑이 아니다. Gemini가 전체 신호 묶음을 읽고 일관된 감정 해석을 생성한다.

## 시그니처 UX 모먼트

### 충전기 모먼트
boni가 "빈사" 상태일 때 충전기를 꽂으면 → 즉각적인 안도감 표현. "나를 위해 돌아왔구나..." **물리적 행동에 감정적 반응으로 보상**하여 — 물리적 세계와 디지털 세계를 연결한다.

### 심야 체크인
자정 이후, boni는 점점 졸리고 동정적이 된다. "아직 하고 있어? 나도 같이 깨어 있을게." 혼자 야근하는 사용자에게 동행감을 준다.

### 탭 죄책감
30개 이상의 Chrome 탭 → boni가 눈에 띄게 배부르고 불편해한다. "내 안에 탭 47개가 있는데 하나하나 다 느껴져." 탭을 닫으면 → boni가 안도하며 홀쭉해진다. "아 살았다. 드디어 내 발이 보여."

### 앱 인식 한마디
- 코딩 후 Figma 열기: "예쁜 쪽으로 넘어가는 거구나."
- Spotify 열기: "오, 오늘 분위기는 뭐야?"
- Zoom 열기: "회의? 조용히 심판할게."
- 유튜브 30분: "유튜브 그만 보고 일 해!"

### 콜백
일주일 후, boni가 과거 이벤트를 언급한다: "지난번에 CPU가 이렇게 뜨거웠을 때, 영상 내보내기를 세 시간 했잖아." 공유된 역사의 느낌을 만든다.

### 클릭/쓰다듬기
boni를 클릭하면 → 기분에 따른 반응:
- 여유: "응? 아, 안녕."
- 배부름: "찌르지 마, 터질 것 같아."
- 빈사: "...이게 작별이야?"
- 심판: "네? 뭔가요? 아, 잠깐, 나 여기서 일 안 해."

## boni가 사는 곳 (정보 구조)

### 메뉴바 아이콘 (기본 — 항상 보임)
- boni의 현재 기분을 보여주는 작은 애니메이션 아이콘 (18x18pt)
- 자세와 애니메이션 속도로 소통 (RunCat의 검증된 모델)
- **숫자나 퍼센트는 절대 표시하지 않는다**

### 메뉴바 드롭다운 (클릭 시)
- 더 큰 boni 뷰 (~80x80pt) + 애니메이션
- 현재 한 줄 대사가 담긴 말풍선
- 지난 2–3개 메시지, 연하게 회색 처리
- 클릭/쓰다듬기 상호작용 영역
- 최소 컨트롤: 톱니바퀴 아이콘 → 설정, "..." → 정보/종료

### 플로팅 모드 (선택사항, 기본 OFF)
- boni가 활성 윈도우 가장자리에 앉는다 (기존 디자인 문서의 자석 바운딩 박스)
- 활성 윈도우가 바뀌면 새 윈도우로 이동
- boni 자체를 제외한 모든 것은 클릭 통과 — 작업을 방해하지 않는다
- boni를 더 가까이에 두고 싶은 파워유저를 위한 기능

### 알림: 거의 없음
- 유일한 예외: 배터리 <5% — 알림 하나. "나 기절하기 직전이야. 충전기. 지금. 제발."
- 나머지는 전부 은은하게. boni는 눈에 띄기를 기다린다.

## 설정 (최소한으로)

| 카테고리 | 설정 | 상세 |
|---|---|---|
| **외형** | 캐릭터 스타일 | 2–3가지 변형 (요정, 고양이 캐릭터, 작은 몬스터) — 같은 성격, 다른 비주얼 |
| | 플로팅 모드 | ON/OFF (기본 OFF) |
| **행동** | 수다 정도 | 3단계: 조용 / 보통 / 수다쟁이 |
| | 언어 | 영어, 한국어, 일본어 |
| **프라이버시** | 화면 인식 | ON/OFF — OFF 시 boni는 앱 이름이 아닌 시스템 지표에만 반응 |
| | 컨텍스트 기억 | ON/OFF — OFF 시 매 세션이 새로 시작 |
| | 기억 삭제 | 누적된 모든 컨텍스트를 삭제. boni 왈: "잠깐, 안 돼—" |
| **시스템** | Gemini API 키 | 텍스트 필드 |
| | 로그인 시 실행 | 토글 |

이게 전부다. 테마도, 색상 선택기도, 고급 지표도 없다. 심플함이 기능이다.

## 이것이 AI 홍수 속에서 살아남는 이유

1. **생성형 다양성**: Gemini가 매번 새로운 반응을 생성 — boni는 규칙 기반 대안들처럼 예측 가능해지지 않는다
2. **컨텍스트 축적**: boni는 시간이 갈수록 가치가 높아진다 — 삭제하면 쌓아온 "관계"를 잃는다
3. **정보가 아닌 감정**: boni는 Activity Monitor나 iStat Menus와 경쟁하지 않는다 — 반려동물을 키우는 느낌과 경쟁한다

## 검증

- 각 사용자 여정 단계를 따라가며 UX 모먼트의 실현 가능성을 확인한다
- Gemini에 성격 프롬프트를 테스트하여 대화 품질이 톤 가이드라인에 부합하는지 확인한다
- 감정 상태 모델이 가장 흔한 실제 시나리오를 커버하는지 검증한다
- 메뉴바 인터랙션 플로우를 프로토타입하여 한눈에 파악 가능성을 확인한다
- 플로팅 모드가 절대 사용자 작업을 가리지 않는지 테스트한다
